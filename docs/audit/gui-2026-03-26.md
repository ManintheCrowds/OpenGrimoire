## 2. Critical user journeys (3–7)

1. **Land → open context graph** — `/` or nav → `/context-atlas` → graph or empty/placeholder. APIs: `GET /api/brain-map/graph`.
2. **Land → visualization** — Home card or nav → `/visualization` → D3 mount. Mostly client-side.
3. **Operator intake (legacy)** — Home card → `/operator-intake` → multi-step form → submit. Depends on Supabase when enabled.
4. **Admin controls** — Nav → `/admin/controls` → global visualization settings.
5. **Capabilities discovery** — Nav → `/capabilities`; API check `GET /api/capabilities` (`e2e/capabilities.spec.ts`).

- **Error / loading:** Context atlas specs expect loading copy and empty states (`e2e/context-atlas.spec.ts`); brain map components should expose test ids (`brain-map-graph`, `brain-map-placeholder-hint`).

Optional manual SR spot-check (NVDA/JAWS/VoiceOver) remains for full AT confidence — see §10.
