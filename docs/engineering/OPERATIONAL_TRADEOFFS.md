# OpenGrimoire — operational tradeoffs

Concise engineering decisions for production and agent integration. Normative HTTP rules remain in [`ARCHITECTURE_REST_CONTRACT.md`](../ARCHITECTURE_REST_CONTRACT.md).

| Topic | Decision |
|-------|----------|
| **Rate limits** | In-memory sliding windows in [`middleware.ts`](../../middleware.ts) are **per Node process**. Multi-instance or serverless replicas need a **shared** limiter (Redis / edge KV) or accept **approximate** limits per instance. **Production scale-out:** prefer **edge/WAF** (CDN rules per path + IP) or **Redis/Upstash** sliding windows so limits survive process boundaries; keep **429** + `Retry-After` semantics where possible. **Client IP for limits:** [`getRateLimitClientIp`](../../src/lib/rate-limit/get-client-ip.ts) uses `X-Forwarded-For` (leftmost) / `X-Real-IP` only when **`OPENGRIMOIRE_TRUST_FORWARDED_IP`** is `1` or `true`, or when **`VERCEL=1`** (platform edge). Otherwise the key is **`unknown`** (one shared bucket per process — limits still apply, but not per spoofed IP). Self-hosted: set the env **and** configure the reverse proxy per [`DEPLOYMENT.md`](../../DEPLOYMENT.md). Applies to survey POST, login POST, **`POST /api/operator-probes/ingest`**, and discovery GETs — each path’s window is independent per process. |
| **Discovery GET limits** | `GET /api/capabilities` and `GET /api/openapi` (+ `.json`) share a generous per-IP limit in middleware (see [ARCHITECTURE_REST_CONTRACT.md](../ARCHITECTURE_REST_CONTRACT.md) § Discovery GET rate limiting). **Stricter** per-key limits for alignment/clarification when API keys are present are **not** implemented in middleware (header parsing cost); prefer **edge/WAF** or a shared rate store in front of those paths in production. |
| **Operator probe ingest limits** | **`POST /api/operator-probes/ingest`** uses the same per-process middleware sliding window as other limited paths (see [ARCHITECTURE_REST_CONTRACT.md](../ARCHITECTURE_REST_CONTRACT.md) § Operator probe ingest rate limiting). **Same scale-out caveat** as **Rate limits**: each replica maintains its own counters unless you add edge/WAF or a shared store. |
| **Operator probe SQLite + replicas** | Probe rows live in the same **SQLite** file as the rest of the app (`data/opengrimoire.sqlite` by default). **Each Node replica** that uses its **own** file has an **independent** `operator_probe_runs` table — not a replicated global log. **TTL deletes** run in-process when **admin list** (`GET /api/admin/operator-probes`) runs and after **ingest** (`purgeExpiredOperatorProbeRuns` in [`operator-probes.ts`](../../src/lib/storage/repositories/operator-probes.ts)); **detail GET** does not purge. **No** scheduled purge. For **replicas > 1** with shared traffic, assume **per-replica** ingest rate limits ([`middleware.ts`](../../middleware.ts)) and plan **edge/shared limiters** or a **scheduled purge** if you need stricter cross-instance behavior — [ARCHITECTURE_REST_CONTRACT.md](../ARCHITECTURE_REST_CONTRACT.md#operator-probe-multi-instance). |
| **Operator probe `raw_blob` at rest** | Optional **`raw_blob`** is stored **inline** in SQLite (ingest cap **512k** chars). Large traces grow the **same** DB file as the rest of the app; there is **no** bundled object store. **Future:** externalize blobs via **ADR** (new URI/key column or URI convention + UI resolution + object lifecycle vs TTL) — [ARCHITECTURE_REST_CONTRACT.md](../ARCHITECTURE_REST_CONTRACT.md#operator-probe-raw-blob-at-rest). |
| **`access_denied` log volume** | **401** + **`invalid_secret`** lines can spike under scanning. Optional **`ACCESS_DENIED_INVALID_SECRET_LOG_PROBABILITY`** and **`ACCESS_DENIED_INVALID_SECRET_PER_IP_COOLDOWN_MS`** in [`access-denial-log.ts`](../../src/lib/observability/access-denial-log.ts) (per-process). **503** misconfigured paths omit `access_denied` where documented (e.g. probe ingest). Details: [OPERATOR_LOG_FIELDS.md](../engineering/OPERATOR_LOG_FIELDS.md). |
| **Survey POST abuse** | `POST /api/survey` is intentionally **unauthenticated** for Sync Session intake. Baseline: middleware rate limit. **Body:** Zod caps **`answers`** at **64** rows (see `surveyPostBodySchema`). **Public internet:** add **Cloudflare Turnstile** (`TURNSTILE_SECRET_KEY` + optional `NEXT_PUBLIC_TURNSTILE_SITE_KEY` for the widget) — enforced when `SURVEY_POST_CAPTCHA_REQUIRED=true` **or** (`NODE_ENV=production` and `TURNSTILE_SECRET_KEY` is set). Server verifies via Turnstile siteverify before DB writes. **Signed bootstrap token:** `SURVEY_POST_REQUIRE_TOKEN=true` requires `SURVEY_POST_BOOTSTRAP_SECRET` and a valid `x-survey-post-token` from **`GET /api/survey/bootstrap-token`** (same-origin UI fetches automatically). **Threat model:** [security/SURVEY_POST_BOOTSTRAP_THREAT_MODEL.md](../security/SURVEY_POST_BOOTSTRAP_THREAT_MODEL.md). See [`docs/agent/SYNC_SESSION_HANDOFF.md`](../agent/SYNC_SESSION_HANDOFF.md). Monitor volume and duplicate fingerprints. **Horizontal replicas (R5):** the same **per-process** sliding window applies to each Node instance; behind a load balancer, clients effectively get roughly the sum of per-replica budgets unless you add a **shared** limiter (Redis/Upstash) or **edge/WAF** rules scoped to `POST /api/survey` (path + IP) so abuse is bounded at the boundary. |
| **Admin moderation PATCH** | [`PATCH /api/admin/moderation/:responseId`](../../src/app/api/admin/moderation/[responseId]/route.ts): body validated with [`moderationPatchBodySchema`](../../src/lib/admin/moderation-patch-body.ts); optional **`notes`** capped at **4000** characters (**400** if over). |
| **Survey POST handler order** | Implementation order in [`survey/route.ts`](../../src/app/api/survey/route.ts): (1) optional `x-survey-post-token` when `SURVEY_POST_REQUIRE_TOKEN` — **401** if invalid; (2) JSON body parse — **400** if not JSON; (3) Zod `surveyPostBodySchema` — **400** (includes **`answers` length** max 64 rows in [`schemas.ts`](../../src/lib/survey/schemas.ts)); (4) optional Turnstile verify when captcha enforced — **400** if token missing/invalid, **503** if captcha required but secret unset; (5) `mapAnswersToSurveyResponse` — **400**; (6) DB write. Agents should read **401** vs **400** vs **503** separately from validation issues. |
| **Scoped API secrets** | Optional **`CLARIFICATION_QUEUE_API_SECRET`** + `x-clarification-queue-key` splits clarification from alignment automation keys. Keep **`ALIGNMENT_CONTEXT_KEY_ALLOWS_SURVEY_READ`** off unless a single key must read survey PII — prefer **`SURVEY_VISUALIZATION_API_SECRET`**. See [`AGENT_INTEGRATION.md`](../AGENT_INTEGRATION.md). **Operator runbook:** [admin/SURVEY_READ_GATING_RUNBOOK.md](../admin/SURVEY_READ_GATING_RUNBOOK.md). |
| **Survey visualization reads** | Tradeoff: **`SURVEY_VISUALIZATION_ALLOW_PUBLIC=true`** (simple public demos) vs **strict** gating (no public reads; admin session or headers only). See [`survey-read-gate.ts`](../../src/lib/survey/survey-read-gate.ts) and [`AGENT_INTEGRATION.md`](../AGENT_INTEGRATION.md). **`NODE_ENV`:** `checkSurveyReadGate` **only** enforces the matrix when **`NODE_ENV=production`**; any other value **skips** the gate (open). Staging with real PII must use **production** `NODE_ENV` if the read gate should apply — [DEPLOYMENT.md](../../DEPLOYMENT.md), [admin/SURVEY_READ_GATING_RUNBOOK.md](../admin/SURVEY_READ_GATING_RUNBOOK.md) § NODE_ENV and staging. |
| **Playwright / E2E env defaults** | [`e2e/helpers/e2e-secrets.ts`](../../e2e/helpers/e2e-secrets.ts) defines predictable literals for **`webServer.env`** when CI/local E2E does not inject secrets. **Not** a production configuration surface — real hosts must set secrets explicitly ([DEPLOYMENT.md](../../DEPLOYMENT.md) § E2E defaults vs production). |
| **Operator auth** | **Single** shared operator password + signed JWT cookie session. **Multi-operator** or multi-device sessions would require a different model (e.g. Lucia + SQLite sessions table) — optional upgrade path. |
| **Capability discovery** | **Hand-maintained** [`GET /api/capabilities`](../../src/app/api/capabilities/route.ts) + entity matrix in [`ARCHITECTURE_REST_CONTRACT.md`](../ARCHITECTURE_REST_CONTRACT.md) vs **generated OpenAPI** (OA-REST-2): lower drift risk from codegen vs ongoing maintenance cost. |
| **Agent-native parity / discovery scores** | Raising scores in the [agent-native audit](../AGENT_NATIVE_AUDIT_OPENGRIMOIRE.md) usually means **harness** and **MCP** work **outside** this repo (thin tools over REST), not additional React surface area. |
| **LLM token budgets** | This app **does not** enforce spend or output-token limits on behalf of callers. Any LLM use is **outside** the OpenGrimoire server (Cursor, harness scripts, local Ollama, etc.). Caller/harness responsibility. Harness env vars and truncation behavior: [MiscRepos `docs/agent/CALLER_SIDE_LLM_BUDGETS.md`](../../MiscRepos/docs/agent/CALLER_SIDE_LLM_BUDGETS.md). Product entry: [`AGENT_INTEGRATION.md`](../AGENT_INTEGRATION.md) § Token budgets. |
| **Agent transcript compaction** | The Next.js app **does not** compact, summarize, or prune **agent chat transcripts** (IDE or harness). **Non-goal** for the server; optional **harness-side** handoff rules: [MiscRepos `docs/agent/HANDOFF_COMPACTION.md`](../../MiscRepos/docs/agent/HANDOFF_COMPACTION.md). |
