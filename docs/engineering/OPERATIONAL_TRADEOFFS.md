# OpenGrimoire â€” operational tradeoffs

Concise engineering decisions for production and agent integration. Normative HTTP rules remain in [`ARCHITECTURE_REST_CONTRACT.md`](../ARCHITECTURE_REST_CONTRACT.md).

| Topic | Decision |
|-------|----------|
| **Rate limits** | In-memory sliding windows in [`middleware.ts`](../../middleware.ts) are **per Node process**. Multi-instance or serverless replicas need a **shared** limiter (Redis / edge KV) or accept **approximate** limits per instance. **Production scale-out:** prefer **edge/WAF** (CDN rules per path + IP) or **Redis/Upstash** sliding windows so limits survive process boundaries; keep **429** + `Retry-After` semantics where possible. **IP trust:** only treat `X-Forwarded-For` as authoritative when the **edge** sets it; otherwise client-supplied headers can bypass naive per-IP limits. |
| **Discovery GET limits** | `GET /api/capabilities` and `GET /api/openapi` (+ `.json`) share a generous per-IP limit in middleware (see [ARCHITECTURE_REST_CONTRACT.md](../ARCHITECTURE_REST_CONTRACT.md) Â§ Discovery GET rate limiting). **Stricter** per-key limits for alignment/clarification when API keys are present are **not** implemented in middleware (header parsing cost); prefer **edge/WAF** or a shared rate store in front of those paths in production. |
| **Survey POST abuse** | `POST /api/survey` is intentionally **unauthenticated** for Sync Session intake. Baseline: middleware rate limit. **Public internet:** add **Cloudflare Turnstile** (`TURNSTILE_SECRET_KEY` + optional `NEXT_PUBLIC_TURNSTILE_SITE_KEY` for the widget) â€” enforced when `SURVEY_POST_CAPTCHA_REQUIRED=true` **or** (`NODE_ENV=production` and `TURNSTILE_SECRET_KEY` is set). Server verifies via Turnstile siteverify before DB writes. **Signed bootstrap token:** `SURVEY_POST_REQUIRE_TOKEN=true` requires `SURVEY_POST_BOOTSTRAP_SECRET` and a valid `x-survey-post-token` from **`GET /api/survey/bootstrap-token`** (same-origin UI fetches automatically). See [`docs/agent/SYNC_SESSION_HANDOFF.md`](../agent/SYNC_SESSION_HANDOFF.md). Monitor volume and duplicate fingerprints. |
| **Survey POST handler order** | Implementation order in [`survey/route.ts`](../../src/app/api/survey/route.ts): (1) optional `x-survey-post-token` when `SURVEY_POST_REQUIRE_TOKEN` â€” **401** if invalid; (2) JSON body parse â€” **400** if not JSON; (3) Zod `surveyPostBodySchema` â€” **400**; (4) optional Turnstile verify when captcha enforced â€” **400** if token missing/invalid, **503** if captcha required but secret unset; (5) `mapAnswersToSurveyResponse` â€” **400**; (6) DB write. Agents should read **401** vs **400** vs **503** separately from validation issues. |
| **Scoped API secrets** | Optional **`CLARIFICATION_QUEUE_API_SECRET`** + `x-clarification-queue-key` splits clarification from alignment automation keys. Keep **`ALIGNMENT_CONTEXT_KEY_ALLOWS_SURVEY_READ`** off unless a single key must read survey PII â€” prefer **`SURVEY_VISUALIZATION_API_SECRET`**. See [`AGENT_INTEGRATION.md`](../AGENT_INTEGRATION.md). |
| **Survey visualization reads** | Tradeoff: **`SURVEY_VISUALIZATION_ALLOW_PUBLIC=true`** (simple public demos) vs **strict** gating (no public reads; admin session or headers only). See [`survey-read-gate.ts`](../../src/lib/survey/survey-read-gate.ts) and [`AGENT_INTEGRATION.md`](../AGENT_INTEGRATION.md). |
| **Operator auth** | **Single** shared operator password + signed JWT cookie session. **Multi-operator** or multi-device sessions would require a different model (e.g. Lucia + SQLite sessions table) â€” optional upgrade path. |
| **Capability discovery** | **Hand-maintained** [`GET /api/capabilities`](../../src/app/api/capabilities/route.ts) + entity matrix in [`ARCHITECTURE_REST_CONTRACT.md`](../ARCHITECTURE_REST_CONTRACT.md) vs **generated OpenAPI** (OA-REST-2): lower drift risk from codegen vs ongoing maintenance cost. |
| **Agent-native parity / discovery scores** | Raising scores in the [agent-native audit](../AGENT_NATIVE_AUDIT_OPENGRIMOIRE.md) usually means **harness** and **MCP** work **outside** this repo (thin tools over REST), not additional React surface area. |
